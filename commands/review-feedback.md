# Review Feedback

レビュー指摘された部分を修正したことを前提に、指摘内容に問題ないか再レビューを行う。

## 目的

レビュー指摘への対応が漏れなく反映されているかを確認し、修正漏れや新たな問題の混入を防ぐ。

## 引数仕様(`$ARGUMENTS`)

- PR 番号(省略可): 指定した場合、その PR のレビューコメントを指摘元とする。省略時の扱いは実行手順の指摘元特定を参照する。

## 実行手順

1. 指摘元を以下の優先順位で決定する。
   1. `$ARGUMENTS` に PR 番号があれば、その PR のレビューコメントを指摘元とする。
   2. なければ、セッション(現在の会話)内にレビュー結果(`/review-branch` 等による指摘)があれば、それを指摘元とする。
   3. どちらも無ければ `gh pr view` で現在のブランチに紐づく PR を特定する。
2. 指摘元に応じて指摘一覧を作る。PR の場合は `gh api` または `gh pr view` でレビューコメント(レビュースレッド)を取得する。セッションの場合は会話中のレビュー指摘をそのまま指摘一覧として使う。
3. 指摘ごとに該当箇所の現在のコードを Read/Grep で確認し、修正が反映されているか判定する。指摘時点からの差分確認が必要な場合、PR コメントであればコメントに紐づくコミット SHA と現在のコードを `git diff` `git log` で比較する(セッション指摘にはコミット SHA が無いため、この比較は行わない)。
4. 指摘ごとに判定結果を報告する。

## 出力形式

以下の列を持つ表で報告する。

`指摘内容 | 該当箇所 | 判定(解消 / 部分解消 / 未対応 / 新規問題あり) | 根拠`

表の後に、未対応または新規問題がある場合は要対応事項として列挙する。
