# Commit Message

ステージ済みの変更(`git diff --staged`)からリポジトリの規約に沿ったコミットメッセージ案を複数提示する。

## 目的

コミット前に、変更内容を正しく要約し、かつリポジトリの直近のコミット規約(文体・粒度・プレフィックス)に沿ったメッセージ案を素早く得る。

## 引数仕様(`$ARGUMENTS`)

- 追加の指示があれば `$ARGUMENTS` として渡される(例: 「英語で」「Conventional Commits 形式で」「1行に収めて」)。
- 引数が空の場合は既定の方針(後述)でメッセージ案を作成する。

## 実行手順

1. `git status --short` でステージ済み・未ステージの変更を確認する。ステージ済みの変更が無ければその旨を報告し、`git add` を促して終了する(自動で `git add` は行わない)。
2. `git diff --staged` でステージ済みの差分を読む。差分が大きい場合はファイル単位で要点を把握する。
3. `git log --oneline -20` で直近のコミット履歴を確認し、文体(日本語/英語)・プレフィックス規約(例: `<対象>: <内容>`)・1行の長さの目安を把握する。
4. `$ARGUMENTS` に追加の指示があれば、それを優先してメッセージ案に反映する。
5. 変更の「何を」ではなく「なぜ」に重点を置いたメッセージ案を **3案** 作成する(粒度や視点を変える: 例. 簡潔な1行のみ / 1行 + 本文で背景説明 / 別の切り口)。
6. 各案について、対応する変更ファイルの一覧を添える。

## 出力形式

```
## コミットメッセージ案

### 案1
<メッセージ>

### 案2
<メッセージ>

### 案3
<メッセージ>

## 対象ファイル
- <path>
- ...
```

## 動作上の注意

- コミットの実行(`git commit`)は行わない。あくまで案の提示に留める。
- 推測でファイル内容を補わない。`git diff --staged` に現れた情報のみを根拠にする。
- 機密情報(トークン・パスワード等)が差分に含まれる場合は警告し、コミットメッセージ案の前に明示する。
