# 観点別 出力フォーマット（Phase 1 共通）

各 Sub Agent は担当観点の評価結果を、評価モード（branch / repo / slice）に応じた以下のフォーマットで返す。重大度の凡例・判断は `severity-criteria.md` に従う。

## 良い点の扱い（共通・必須）

- **良い点は全モード（branch / repo / slice）で必須**。観点に照らして優れている点を記載する。
- **指摘より先に記載する**（読み手はまず良い点を把握してから指摘を読む）。
- 該当が無ければ省略せず「特になし」と明記する。

## 重大度の凡例（共通）

- 🔴 Critical: マージ前に必ず修正
- 🟠 High: 修正強く推奨
- 🟡 Medium: 検討推奨
- 🟢 Low / Info: 参考情報

## review-branch（差分レビュー・第1パス）

```
## [観点名] レビュー（第1パス）

### 良い点
（観点に照らして優れている点を簡潔に。**必須**。無ければ「特になし」）

### 指摘事項
- 🔴/🟠/🟡/🟢 [ファイル:行] 指摘内容
  - 根拠: なぜ問題か
  - 提案: 具体的な改善案（コード例があればなお良い）

### この観点でのカバー範囲
- 確認したファイル: ...
- 確認できなかったもの・前提: ...
```

## review-repo（分類 × 観点スキャン・Phase 1 共通）

```
## [分類名] / [観点名] スキャン

### 概況
（この分類・観点に対する1〜2段落の総評）

### 良い点
（この分類・観点で優れている点を簡潔に。**必須**。無ければ「特になし」）

### 主要発見（上位 $TOP 件、または少数なら全件）
| 順位 | 対象 | 指標値 | 重大度 | 一言コメント |
|---|---|---|---|---|
| 1 | path/to/file | 行数 1200 / 変更頻度 45 | 🔴 Critical | ... |

### 分布の特徴
- ...

### 推奨アクション（この観点で）
- 短期 / 中期 / 長期

### 制約・前提
- 目視推定の箇所、ツール未使用、誤検知要因
```

> ⚠️ 補助観点（マトリクスの ⚠️）は出力を簡略化する: 概況1〜2段落 + 主な発見3件まで（テーブル省略可）。詳細は SKILL.md の「✅（主要）と ⚠️（補助）の運用差」。

## review-slice（スライス評価・Phase 1 共通）

```
## [観点名] スライス評価

### 概況
（スライス全体に対するこの観点での総評）

### 良い点
（スライスに照らして優れている点を簡潔に。**必須**。無ければ「特になし」）

### 主要発見
| ファイル | 重大度 | 指摘内容 | 根拠 | 提案 |
|---|---|---|---|---|

### スライス特有の論点
（レイヤー越え・コンテキスト越え等、スライスならではの発見）

### 制約・前提
- 動的解決の影響、追跡できなかった範囲
```

> シンボル起点（`path::symbol`）の場合、各対象に **経路上 / 起点同居 / 依存先** のタグを付す（例: `order_service.py::OrderService.checkout（経路上）` / `order_service.py 他メソッド（起点同居）` / `payment_gateway.py（依存先）`）。起点同居は責務集中の判断材料で、スライス本体（経路上＋依存先）の指摘とは区別する。
